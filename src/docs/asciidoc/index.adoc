[[overview]]
== 개요

Plantly 서비스의 REST API 명세서입니다.
모든 응답은 아래 공통 래퍼(`ApiResponse`) 형태로 내려갑니다.

[[overview-response-wrapper]]
=== 공통 응답 형식

[cols="2,1,5"]
|===
| 필드 | 타입 | 설명

| `success` | `boolean` | 요청 성공 여부
| `message` | `string` | 성공 메시지 (성공 시, 없을 수 있음)
| `data`    | `object` | 응답 데이터 (성공 시, 없을 수 있음)
| `error`   | `string` | 에러 메시지 (실패 시에만 존재)
|===

NOTE: `ApiResponse` 는 `@JsonInclude(NON_NULL)` 이므로 값이 `null` 인 필드는 JSON 응답에서 생략됩니다.
예를 들어 회원가입 성공 응답에는 `data`/`error` 가 나타나지 않습니다.

[[overview-auth]]
=== 인증 / 보안 공통 규약

세션 기반 인증을 사용합니다. 인증 메커니즘은 전역 공통 규약이므로 각 API 문서에 반복하지 않고 여기에 한 번만 명시합니다.

* *로그인*: `POST /api/v1/auth/login` 성공 시 `JSESSIONID` 세션 쿠키가 발급됩니다.
  이후 인증이 필요한 요청에는 이 세션 쿠키가 자동으로 동봉되어야 합니다.
* *CSRF*: 상태를 변경하는 모든 요청(`POST`, `PATCH`, `PUT`, `DELETE`)은 CSRF 보호를 받습니다.
  먼저 `GET /api/v1/auth/csrf` 로 토큰을 발급받은 뒤, 발급된 값을 `X-XSRF-TOKEN` 요청 헤더에 담아 보내야 합니다.
  누락 시 `403` 이 반환됩니다.
* *인증 실패*: 미인증 상태로 보호 자원에 접근하면 `401`, 권한이 부족하면 `403` 이 반환됩니다.

[cols="2,1,5"]
|===
| 항목 | 위치 | 설명

| `JSESSIONID` | Cookie | 로그인 시 발급되는 세션 쿠키 (인증 필요 API 필수)
| `X-XSRF-TOKEN` | Header | CSRF 토큰 (변경 요청 필수)
|===

정책/유효성 위반 시 내려갈 수 있는 `error` 메시지는 다음과 같습니다.

[cols="3,5"]
|===

| 레퍼런스 이미지 장수 초과 | 현재 등급에서 등록 가능한 레퍼런스 이미지 장수를 초과했습니다.
| 상세 이미지 장수 초과 | 현재 등급에서 등록 가능한 상세 이미지 장수를 초과했습니다.
| 갤러리 이미지 타입 오류 | 갤러리에는 상세 이미지(DETAIL)만 등록할 수 있습니다.
| 존재하지 않는 카테고리 ID | 존재하지 않는 카테고리가 포함되어 있습니다.
| 존재하지 않는 인증 ID | 존재하지 않는 인증이 포함되어 있습니다.
| 존재하지 않는 국가 ID | 존재하지 않는 국가가 포함되어 있습니다.
| 존재하지 않는 국내 지역 ID | 존재하지 않는 국내 지역이 포함되어 있습니다.
| 존재하지 않는 산업군 ID | 존재하지 않는 산업군이 포함되어 있습니다.
|===

[[company-search]]
=== 회사 목록/검색 (공개)

회사를 검색·브라우즈합니다. 누구에게나 열려 있는 공개 API 로, 인증이 필요 없습니다.

검색은 세 축으로 조합됩니다(모두 선택적, 지정한 것만 AND 로 적용):

* *통합 키워드* (`keyword`): 회사명·소개·레퍼런스·설비·소재 등 전체 텍스트에 부분일치. 공백으로 나눈 여러 토큰은 모두 포함(AND).
* *고급검색* (`companyName`/`content`/`reference` 등): 지정한 필드에만 부분일치.
* *패싯* (`certificationIds`/`industryIds`/`categoryIds`): 선택한 ID 중 하나라도 보유한 회사(차원 내 OR). 카테고리는 대분류를 고르면 그 후손 서브트리까지 매칭됩니다.

정렬은 스팟라이트 → 추천 → 최신 순으로 고정이며, 결과는 요약 카드 형태로 페이징됩니다.
